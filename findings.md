# Findings & Decisions

## Requirements
- 新建一个完整的个人作品集网站项目。
- 搜集市面上已有的优秀个人作品集网站作为参考。
- 结合用户自己的项目内容和期望风格进行设计与实现。
- 最终产出应是可访问、可迭代、适配桌面与移动端的网站。
- 主要定位：视觉 / 创意。
- 风格起点：清晰产品感。
- 素材状态：已有 3–6 个完整项目。
- 新增内容结构要求：作品将按 5–6 个游戏主题分类，每个主题下继续细分多个项目；大多数项目以网页长图为主要展示媒介。

## User Inputs Needed
- 更具体的职业称谓、擅长领域与目标受众。
- 中英文姓名、个人简介、头像或个人视觉素材。
- 3–6 个重点项目：背景、职责、过程、成果、图片/视频、可公开链接。
- 喜欢与不喜欢的网站、字体、颜色、动效和整体气质。
- 功能需求：语言、博客、CMS、简历下载、联系表单、统计等。
- 域名、部署与后续维护偏好。

## Technical Decisions
| Decision | Rationale |
|----------|-----------|
| 暂不选择框架 | 内容更新、上线方式和交互复杂度会影响技术选型 |
| 参考库至少覆盖不同类型作品集 | 避免只研究同一种视觉流派，保证方案有对照依据 |
| 首版平衡视觉冲击与案例可读性 | 用户属于视觉 / 创意方向，但明确偏好清晰产品感 |
| 方向原型使用 Vite 8.1.5 + 原生 ES Modules | 3–6 个静态案例无需引入重型框架，构建和部署更简单；版本已于 2026-07-16 通过 npm registry 核验 |
| 首版用数据文件管理项目内容 | 真实项目到位后可集中替换，避免散落修改页面结构 |
| 使用暖灰、近黑与项目主题色 | 保持清晰专业，同时让视觉项目拥有独立识别 |

## Visual/Browser Findings
- 当前已完成在线状态与页面元信息核验；视觉与交互逐站检查将在参考库第二轮完成。
- 2026-07-16 用户明确指定《明日方舟：终末地》官网 `https://endfield.hypergryph.com/#notice` 作为新版视觉方向参考，关键词为“简约、帅气、几何化”。
- 参考站公开 HTML 显示其为响应式 Next.js 页面，分别以 2560×1440 横屏与 1080×1920 竖屏为设计基准；本项目只借鉴工业科幻的信息秩序、斜切几何、黑白高对比和警示色节奏，不复用游戏品牌、素材或具体界面。
- 当前会话未暴露可视化浏览器连接，因此参考站尚未完成截图式视觉核验；先以用户给出的风格关键词和公开结构作为方向依据，本地页面完成后再做可用范围内的视觉验证。
- 参考站公开 CSS 可确认主要界面色为 `#191919`、灰白层级与 `#fffa00` 信号黄；标题使用宽体粗体，正文使用 Regular / Medium 字重。
- 参考站用细边框、黑白反相、`clip-path: polygon(...)` 切角和短距离位移制造机械感。个人作品集将把这些规律简化为原创的“静默网格 / 高压信号”视觉系统。
- 用户反馈上一版“页面太空、缺少设计感”，明确要求加入几何线条、秩序性、规范性与理性美；因此页面信息架构升级为“主题索引 → 项目矩阵 → 长图档案模块”，不再以单页大留白首屏为主叙事。
- 参考站公开数据使用 `bulletins` 和 `tab`（`notices` / `events` / `news`）组织多类内容，并用 `sticky` 区分重点项；本项目将对应抽象为主题导航、项目数量和重点档案。
- 2026-07-16 对 `https://arocksworld.com/` 的公开资源核验：入口页先显示全屏 loading 层和顶部进度条，再加载桌面/移动专用 CSS 与 JS。
- AROCK 的公开 CSS/JS 可见交互规律：元素进入/离开使用 clip-path、opacity 与 translate3d；悬停时上下两层文字反向切换；页面监听 pointermove、pointerenter/leave、wheel、keydown 和 touch；媒体层使用 opacity、scale、parallax 与 requestAnimationFrame 平滑插值。
- 本项目将这些规律抽象为原创交互：首屏系统启动序列、主题切换的遮罩/裁切过渡、项目卡片的指针倾斜与标签切换、长图进入视口的扫描线和案例弹层的层级转场；全部保留 reduced-motion 降级。

## 2026-07-19 全站高级感交叉审查
- worker 只读审查与 lead 复核一致：首屏 1800:753 品牌展示窗、分类切换、详情直达、系统鼠标和原图查看器已经成熟，应该保持不动。
- 当前最高优先级问题是 `display: long` 作品仍被压进固定高度 `object-fit: contain` 画框，极端长图在 16:9 视口内会缩成几十像素宽的细条，大量背景成为无效阅读面积。
- 解决口径：长图仍保持同行统一高度与绝不裁切，但画框内部改为自然宽度纵向滚动；普通画面继续完整 contain；点击仍进入现有原图查看器。
- 21 张项目封面与首屏品牌封皮存在多张 1.4–3MB PNG，首屏与默认分类加载过重；使用同尺寸 AVIF 代理替换展示路径，不修改原始素材。
- 全站键盘焦点不能再使用低对比的动态主题色；浅色区使用 ink，深色区局部覆写为 paper/accent。
- 归档区仍残留持续扫描线、跳动信号条和重复伪参数标签，与最新首屏的克制编辑感不一致；终检采用减法，只保留静态编号、边界与必要 hover 反馈。
- 2026-07-19 目视检查重新生成的 `ro-dungeon-collab` 与 `xindong-anniversary` 16:9 代理：前者以原画边缘浅蓝灰补齐上下空间，后者按比例轻微缩放并保留原图极细黑边；均未裁切主体，封面内容和文字完整。
- 第三轮 worker 运行时复测确认：`ro-chuxin-cbt` 与 `yise-ss3` 的 PC / Mobile FULL PAGE 均进入 document 滚动模式；SS3 两个 frame 同高 384px，`overflow-y:auto`、图片 `position:static` / `width:100%` / `height:auto`，无 `object-fit:cover` 与裁切。
