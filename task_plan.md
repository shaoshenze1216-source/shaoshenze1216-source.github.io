## Decisions Made
| Decision | Rationale |
|----------|-----------|
| 先研究再设计，不直接套模板 | 作品集需要由个人定位和项目叙事驱动，避免做成通用简历页 |
| 研究与实现分阶段 | 便于先确认方向，再投入视觉与开发成本 |
| 暂不锁定技术栈 | 用户尚未说明维护、部署和内容更新要求 |
| 方向原型采用 Vite + 原生 JavaScript | 当前内容规模小、交互可控，避免过早引入框架复杂度；正式需求变化时再评估 |
| 用四个占位案例先验证结构 | 在不假装拥有真实用户内容的前提下，先交付可运行方向原型 |
| 案例媒体采用真实分组而非固定数量 | 原始项目的 PC、Mobile、状态图数量差异很大；只渲染存在的组，避免伪造内容 |
| Land-book 只转译作品优先与稳定画幅逻辑 | 保留本项目理性几何语言，不复制其商业导航、品牌和卡片外观 |
| PROFILE 作为作品与方法之间的独立章节 | 先用作品建立能力证据，再补充个人背景与经历，最后说明工作方法并给出联系入口 |
| 一级归档固定为 5 个用户指定类别 | 主游戏保留直接项目；“其他”按七个一级文件夹聚合，避免继续暴露多余层级 |
| 项目封面与作品内容使用两套资源口径 | 21 张真实项目封面只用于 16:9 外部卡片，不进入 163 张作品统计 |
| 详情采用来源分段的固定角色网格 + 原图查看器 | 同一来源内画幅和列宽一致，不使用 dense 回填或序号取模；需要阅读细节时再进入可缩放原图层 |
