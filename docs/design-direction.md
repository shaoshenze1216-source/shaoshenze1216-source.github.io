# 设计方向 v0.4

## 核心概念

**理性档案 / 几何秩序。**

用密集、可追踪的档案系统承载 5–6 个游戏主题及其项目；几何线条、编号、坐标和主题信号色共同建立理性秩序。整体借鉴工业科幻界面的规范感，但不复制任何游戏品牌、素材或具体组件。

## 体验原则

1. **五秒内识别身份**：首屏直接出现角色、能力和个人定位。
2. **几何服务信息**：切角、坐标、十字标和网格都承担层级或反馈，不做无意义装饰。
3. **主题先于项目**：先选择游戏主题，再浏览项目，避免大量项目在同一层级竞争。
4. **长图是主媒介**：项目卡片和案例详情均为网页长图预留纵向比例与连续阅读结构。
5. **动效像机械响应**：使用扫描、裁切、短距离位移和精确的进入节奏，避免柔软漂浮。

## 视觉语言

- 底色：冷灰白 `#e9e9e4`，减少纸张与生活方式气质。
- 文字：深黑 `#101210`，高对比、偏工业。
- 强调色：信号黄为默认状态；不同游戏主题可分配一个受控信号色，用于导航和状态反馈。
- 标题：`Barlow Condensed`，窄体、粗字重，建立技术与速度感。
- 正文：`IBM Plex Sans`，保留清晰的功能信息与中文回退。
- 构图：12 栏主网格、1px 分隔线、档案表格和纵向长图，不使用无信息的大面积留白。
- 几何：斜切角、十字定位标、坐标文本、扫描线与轴线。
- 质感：极轻网格和噪点；禁止圆角卡片、柔和阴影和多色渐变。

## 页面结构

1. 系统化顶部导航与可用状态
2. 紧凑身份主张 + CSS 几何索引装置
3. 6 个游戏主题的一级导航
4. 主题下的项目矩阵与网页长图预览
5. 黑底方法区与工作流程
6. 主题信号色联系区
7. 全屏案例详情：角色、范围、问题、判断、结果与多张长图模块

## 响应式原则

- 桌面：以 1440px 横屏为主要检查基准，首屏为 7:5 分栏。
- 平板：项目保持两列，首屏图形缩小但不隐藏。
- 手机：回到单列；保留切角、编号和十字标，减少非关键信息。
- 尊重 `prefers-reduced-motion`，禁用扫描和位移动画。

## 交互语言

- 首次进入：短促的系统启动与进度序列，建立“进入档案”的状态感。
- 主题切换：完整遮罩覆盖后更新内容，再退场，避免突然换色和布局闪烁。
- 指针反馈：桌面精细指针下使用状态光标；选择、打开、观察和关闭各有明确标签。
- 项目浏览：卡片按指针位置产生小幅 3D 倾斜，内部网格随滚动轻微视差，操作标签只在 hover 出现。
- 案例进入：长图模块按顺序裁切进入，强化纵向档案的阅读层级。
- 动效统一使用 `cubic-bezier(.16, 1, .3, 1)` 为主缓动，保持快速响应与克制回弹。
- 触屏、粗指针与 `prefers-reduced-motion` 环境不启用自定义光标、倾斜或视差。

## 正式版待确认

- 真实姓名、职业称谓、一句话定位和城市。
- 5–6 个游戏主题的正式名称、排序和主题色。
- 每个主题下的项目名称、排序、网页长图和过程素材。
- 每个项目的可公开成果、角色、时间和量化结果。
- 是否需要双语、简历下载、CMS、统计与独立项目 URL。
- 域名、部署平台和后续维护方式。
